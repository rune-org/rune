# Scryb, generador de documentación con IA

Scryb genera documentación Markdown para un flujo de trabajo guardado.

Úsalo cuando necesites explicar qué hace un flujo, cómo está conectado o qué debe saber otro compañero antes de editarlo.

## Generar documentación del flujo

1. Abre un flujo de trabajo guardado en el lienzo.
2. Abre el panel de documentación de Scryb.
3. Elige el público objetivo si se te pide.
4. Haz clic en **Generar**.
5. Revisa la documentación generada.
6. Previsualiza o descarga el archivo Markdown.

## Cuándo Scryb es más útil

Usa Scryb para:

- Traspasos entre compañeros.
- Explicar un flujo antes de una revisión.
- Capturar la intención después de que Smith genera un borrador.
- Documentar flujos de producción.
- Crear un punto de partida para manuales de operaciones.

## Antes de compartir la documentación generada

Lee la salida antes de compartirla.

Verifica que:

- El propósito del flujo es correcto.
- Las descripciones de los nodos coinciden con el lienzo real.
- Los detalles de credenciales y servicios no están sobrecompartidos.
- Cualquier advertencia o paso manual es claro.

## Problema común

Si Scryb te pide guardar primero el flujo, guárdalo e inténtalo de nuevo.

Scryb necesita un ID de flujo guardado antes de poder generar documentación.
