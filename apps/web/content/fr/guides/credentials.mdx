# Gérer les identifiants

Les identifiants permettent aux workflows de se connecter à des API et services privés sans placer de secrets directement dans le graphe du workflow.

## Quand vous avez besoin d'un identifiant

Vous avez besoin d'un identifiant lorsqu'un nœud doit prouver son identité à un autre service.

Exemples :

- Appeler une API privée.
- Envoyer un e-mail authentifié.
- Se connecter à un fournisseur OAuth.
- Exécuter une intégration nécessitant un compte.

Vous n'avez généralement pas besoin d'identifiants pour les API publiques telles que le point de terminaison GitHub Zen du démarrage rapide.

## Créer un identifiant

1. Ouvrez **Identifiants**.
2. Cliquez sur **Ajouter un identifiant**.
3. Choisissez le type d'identifiant.
4. Saisissez les valeurs requises.
5. Enregistrez ou connectez l'identifiant.

Rune stocke les valeurs sensibles de manière sécurisée et permet aux workflows de référencer l'identifiant par son nom.

## Utiliser un identifiant dans un workflow

1. Ouvrez le workflow sur le canevas.
2. Sélectionnez le nœud qui a besoin d'un accès.
3. Choisissez l'identifiant dans les paramètres du nœud.
4. Enregistrez le workflow.
5. Lancez une exécution de test.

## Partager avec précaution

Les identifiants peuvent être partagés afin que vos coéquipiers puissent exécuter les workflows qui en dépendent.

Avant de partager, confirmez :

- Que le coéquipier doit être autorisé à utiliser ce compte ou cette clé.
- Que l'identifiant ne dispose que des autorisations dont le workflow a besoin.
- Que supprimer ou modifier l'identifiant affectera les workflows qui l'utilisent.

## Identifiants OAuth

Pour les identifiants OAuth, Rune vous redirige vers le fournisseur, vous approuvez l'accès, et Rune stocke les jetons obtenus pour les runs de workflow.

En cas d'échec OAuth, vérifiez :

- Que le compte du fournisseur dispose des autorisations pour les portées demandées.
- Que l'identifiant a été reconnecté après un changement de portées.
- Que l'administrateur de votre espace de travail a correctement configuré OAuth pour ce fournisseur.

## Supprimer des identifiants

Avant de supprimer un identifiant, Rune vérifie si des workflows le référencent.

Si un workflow utilise toujours l'identifiant, sa suppression peut faire échouer les runs futurs.
