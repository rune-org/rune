# Modèles

Les modèles sont des points de départ de workflow réutilisables.

Utilisez un modèle lorsque vous voulez une structure fonctionnelle plus rapidement qu'en construisant chaque nœud à partir de zéro.

## Parcourir les modèles

Ouvrez **Modèles** pour rechercher ou parcourir les modèles de workflow disponibles.

Un modèle peut inclure :

- Une structure de workflow suggérée.
- Des nœuds et des connexions prédéfinis.
- Des notes sur ce qu'il faut personnaliser.
- Les identifiants ou services requis.

## Utiliser un modèle

1. Ouvrez **Modèles**.
2. Choisissez un modèle correspondant à votre objectif.
3. Examinez les détails.
4. Cliquez sur **Utiliser**.
5. Enregistrez le nouveau workflow.
6. Remplacez les valeurs d'exemple, les identifiants et les messages.
7. Lancez une exécution de test.

## Personnaliser avant de vous y fier

Les modèles sont des points de départ, pas des garanties.

Avant d'en utiliser un pour un travail réel :

- Renommez le workflow.
- Confirmez que chaque nœud fait ce que vous attendez.
- Ajoutez ou mettez à jour les identifiants.
- Exécutez avec des données de test sûres.
- Inspectez la sortie d'exécution.

## Quand partir d'un modèle

Utilisez les modèles lorsque :

- Vous reconnaissez le schéma du workflow.
- Le modèle fait gagner du temps de configuration.
- Vous voulez apprendre à partir d'un exemple connu.

Partez de zéro lorsque le workflow est petit ou très spécifique.
