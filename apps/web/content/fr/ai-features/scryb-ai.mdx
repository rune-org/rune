# Scryb, générateur de documentation IA

Scryb génère une documentation Markdown pour un workflow enregistré.

Utilisez-le lorsque vous devez expliquer ce que fait un workflow, comment il est câblé ou ce qu'un autre coéquipier doit savoir avant de le modifier.

## Générer la documentation d'un workflow

1. Ouvrez un workflow enregistré sur le canevas.
2. Ouvrez le panneau de documentation Scryb.
3. Choisissez le public cible si vous y êtes invité.
4. Cliquez sur **Générer**.
5. Examinez la documentation générée.
6. Prévisualisez ou téléchargez le fichier Markdown.

## Quand Scryb est le plus utile

Utilisez Scryb pour :

- Les passations entre coéquipiers.
- Expliquer un workflow avant une revue.
- Capturer l'intention après que Smith a généré un brouillon.
- Documenter des workflows de production.
- Créer un point de départ pour des runbooks.

## Avant de partager la documentation générée

Lisez la sortie avant de la partager.

Vérifiez que :

- L'objectif du workflow est correct.
- Les descriptions des nœuds correspondent au canevas réel.
- Les détails d'identifiants et de services ne sont pas trop divulgués.
- Les éventuelles réserves ou étapes manuelles sont claires.

## Problème courant

Si Scryb vous demande d'enregistrer d'abord le workflow, enregistrez-le et réessayez.

Scryb a besoin d'un identifiant de workflow enregistré avant de pouvoir générer la documentation.
