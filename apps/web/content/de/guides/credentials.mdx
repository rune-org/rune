# Zugangsdaten verwalten

Zugangsdaten ermöglichen es Workflows, sich mit privaten APIs und Diensten zu verbinden, ohne Geheimnisse direkt in den Workflow-Graphen zu schreiben.

## Wann du Zugangsdaten brauchst

Du brauchst Zugangsdaten, wenn ein Knoten sich gegenüber einem anderen Dienst ausweisen muss.

Beispiele:

- Eine private API aufrufen.
- Eine authentifizierte E-Mail senden.
- Eine Verbindung zu einem OAuth-Anbieter herstellen.
- Eine Integration ausführen, die ein Konto benötigt.

Für öffentliche APIs wie den GitHub-Zen-Endpunkt im Schnellstart brauchst du normalerweise keine Zugangsdaten.

## Zugangsdaten erstellen

1. Öffne **Zugangsdaten**.
2. Klicke auf **Zugangsdaten hinzufügen**.
3. Wähle den Typ der Zugangsdaten.
4. Gib die erforderlichen Werte ein.
5. Speichere oder verbinde die Zugangsdaten.

Rune speichert sensible Werte sicher und lässt Workflows die Zugangsdaten über den Namen referenzieren.

## Zugangsdaten in einem Workflow verwenden

1. Öffne den Workflow auf dem Canvas.
2. Wähle den Knoten aus, der Zugriff benötigt.
3. Wähle die Zugangsdaten in den Knoteneinstellungen aus.
4. Speichere den Workflow.
5. Führe einen Test aus.

## Sorgfältig teilen

Zugangsdaten können geteilt werden, damit Teammitglieder Workflows ausführen können, die davon abhängen.

Vor dem Teilen bestätigen:

- Das Teammitglied sollte berechtigt sein, dieses Konto oder diesen Schlüssel zu verwenden.
- Die Zugangsdaten haben nur die Berechtigungen, die der Workflow benötigt.
- Das Löschen oder Ändern der Zugangsdaten wirkt sich auf Workflows aus, die sie verwenden.

## OAuth-Zugangsdaten

Für OAuth-Zugangsdaten leitet Rune dich zum Anbieter weiter, du genehmigst den Zugriff, und Rune speichert die resultierenden Tokens für Workflow-Läufe.

Wenn OAuth fehlschlägt, prüfe:

- Das Anbieterkonto hat Berechtigung für die angeforderten Scopes.
- Die Zugangsdaten wurden nach Scope-Änderungen erneut verbunden.
- Der Workspace-Administrator hat OAuth für diesen Anbieter korrekt konfiguriert.

## Zugangsdaten löschen

Bevor du Zugangsdaten löschst, prüft Rune, ob Workflows sie referenzieren.

Wenn ein Workflow die Zugangsdaten noch verwendet, kann ihr Löschen dazu führen, dass künftige Läufe fehlschlagen.
