# Authentifizierung

Diese Referenzseite ist für entwicklerseitige Authentifizierungsdetails reserviert.

Für das Benutzer-Onboarding beginne mit [Erste Schritte](/docs/getting-started).

## Benutzeranmeldung

Rune-Benutzer melden sich über die App an. Je nach Workspace-Konfiguration kann die Authentifizierung passwortbasierten Zugang, SAML oder von Zugangsdaten genutzte OAuth-verbundene Dienste umfassen.

## Zugangsdaten sind getrennt

Die App-Authentifizierung ist nicht dasselbe wie Workflow-Zugangsdaten.

- App-Authentifizierung steuert, wer Rune nutzen kann.
- Workflow-Zugangsdaten steuern, auf welche externen Systeme ein Workflow zugreifen kann.

Für benutzerseitige Zugangsdaten-Anleitungen, siehe [Zugangsdaten](/docs/guides/credentials).
