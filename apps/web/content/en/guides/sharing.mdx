# Sharing

Sharing lets teammates view, edit, or run workflows without copying them.

## Share a workflow

1. Open **Workflows**.
2. Find the workflow.
3. Open the share action.
4. Choose the teammate and role.
5. Save the share.

## Roles

Rune supports user-level workflow roles such as:

- **Viewer:** can view the workflow.
- **Editor:** can view, edit, and run the workflow.

Use the smallest role that lets the teammate do their job.

## Shared credentials

A shared workflow may still need credentials.

If a teammate can open a workflow but cannot run it, check whether the required credential is available to them.

Credentials should be shared more carefully than workflows because they represent access to external systems.

## Before sharing

Ask:

- Should this teammate see the workflow logic?
- Should this teammate be able to change it?
- Does the workflow use sensitive data or credentials?
- Will edits affect other users?

## Revoke access

If someone no longer needs access, remove them from the workflow share list.

If a credential should no longer be available, revoke or rotate it separately.
