# Scryb AI Documentation Generator

Scryb generates Markdown documentation for a saved workflow.

Use it when you need to explain what a workflow does, how it is wired, or what another teammate should know before editing it.

## Generate workflow docs

1. Open a saved workflow on the canvas.
2. Open the Scryb documentation panel.
3. Choose the target audience if prompted.
4. Click **Generate**.
5. Review the generated documentation.
6. Preview or download the Markdown file.

## When Scryb is most helpful

Use Scryb for:

- Handoffs between teammates.
- Explaining a workflow before review.
- Capturing intent after Smith generates a draft.
- Documenting production workflows.
- Creating a starting point for runbooks.

## Before sharing generated docs

Read the output before you share it.

Check that:

- The workflow purpose is correct.
- Node descriptions match the actual canvas.
- Credential and service details are not overshared.
- Any caveats or manual steps are clear.

## Common issue

If Scryb asks you to save the workflow first, save the workflow and try again.

Scryb needs a saved workflow ID before it can generate documentation.
