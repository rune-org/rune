# Authentication

This reference page is reserved for developer-facing authentication details.

For user onboarding, start with [Getting Started](/docs/getting-started).

## User sign-in

Rune users sign in through the app. Depending on workspace setup, authentication may include password-based access, SAML, or OAuth-connected services used by credentials.

## Credentials are separate

App authentication is not the same as workflow credentials.

- App authentication controls who can use Rune.
- Workflow credentials control what external systems a workflow can access.

For user-facing credential guidance, see [Credentials](/docs/guides/credentials).
